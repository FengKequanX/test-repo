## Work Performed

### 根本原因定位

通过 Jetson 远程诊断，定位了分割掩码绘制的核心问题：

1. **Bbox 格式错误** — ONNX 输出是 `(x1, y1, x2, y2)`，不是 `(cx, cy, w, h)`。代码错误地做了 `xywh→xyxy` 转换，把每个坐标值都当成中心点处理，导致框大了一倍。

2. **Mask 生成算法错误** — 之前在矩阵乘法前对系数应用了 sigmoid，然后 resize 到 bbox 大小，导致掩码几何畸变（92×98 → 785×414 拉伸 8 倍）。

3. **Sigmoid 使用位置错误** — 官方算法不使用 sigmoid，直接在 0 处阈值。

### 修复内容

#### `test_image_v4.py` 修复：
- 删除了 `xywh2xyxy()` 函数 — 不再需要转换
- 修正 `scale_boxes_to_orig()` — 直接乘以 gain（已经是 xyxy 格式）
- 重写 `generate_mask()` — 采用官方 `process_mask_native` 算法
  - `mask_coef @ proto_flat` → upsample 到原图 → 原图坐标 crop → `>0` 阈值
- 移除 sigmoid 应用

#### `yolo_detector.py` 修复：
- `preprocess()`: letterbox → 直接 resize（匹配 rect=False 训练配置）
- `postprocess()`: xyxy 格式直接逆变换，移除 letterbox 偏移处理
- `nms()`: 改为 class-aware NMS（同类重叠才抑制，跨类不抑制）
- `class_names`: 修正为 `{0: "port", 1: "switch"}`（之前错误）
- `visualize_detections()`: 重写 mask 生成为官方算法
